01

开始之前

这个工具能做什么

简单说，这个工具帮你自动从电脑上的微信里读取聊天记录，然后用 AI 帮你分析、整理成法院可用的证据材料。

具体来说：

  • 自动读取聊天记录 — 不需要手动截图、不需要一条一条复制粘贴，工具直接从微信的本地数据库读取
  • 支持所有消息类型 — 文字、图片、语音、视频、转账记录、红包、文件，全部都能提取
  • 语音自动转文字 — 客户发的语音消息会自动识别转为文字，方便阅读和作为书面证据
  • AI 智能分析 — 帮你从大量聊天中找出关键证据，分析争议焦点，评估证据链完整性
  • 一键生成证据清单 — 直接输出符合法院要求格式的 Word 文档，包含完整的证据编号、来源、证明目的等

你需要准备什么

  1. 一台装了微信的电脑 — 就是你平时用来和客户聊微信的那台电脑（Windows 或 Mac 都行）
  2. 微信处于登录状态 — 工具运行时，微信需要是打开的、已登录的状态
  3. 网络连接 — AI 分析功能需要联网（提取聊天记录本身不需要）

数据安全说明

你的数据安全是第一位的。

所有聊天记录都在你自己的电脑上处理，不会上传到任何服务器。

工具只是"读取"微信的本地数据，不会修改、删除微信的任何数据。

使用 AI 分析时，只发送文字内容给 AI 服务，图片和视频原文件不会发送。

02

安装指南

别担心，虽然这是一个命令行工具，但安装过程其实很简单。按照下面的步骤，一步一步来就行。

第 1 步：安装 Python

Python 是运行本工具需要的环境，类似于"运行引擎"。

  1. 打开浏览器，访问 python.org

  2. 点击页面上方的 Downloads，再点 Download Python 3.12

  3. 下载完成后运行安装程序

最重要的一步：安装界面底部有一个 "Add Python to PATH" 的勾选框，一定要勾上！

如果忘记勾选，后面所有步骤都会出错。如果已经安装但忘记勾了，卸载重装一次就行。

  1. 勾选后点击 Install Now，等待安装完成

  2. 验证安装是否成功：按键盘 Win + R，输入 cmd，按回车打开命令提示符，然后输入：

    python --version

    看到类似 Python 3.12.x 的输出就说明安装成功了。

第 2 步：下载证据助手

把项目文件夹放到你想要的位置，比如 D:\wechat_evidence_agent\。

第 3 步：安装依赖

"依赖"就是工具运行需要的其他软件包，一条命令就能全部装好。

cd D:\wechat_evidence_agent
pip install -e ".[all]"

安装过程可能需要几分钟，取决于你的网速。如果看到红色的 ERROR 字样，大多数情况是网络问题，重新运行一次就好。

第 4 步：配置

工具需要一个配置文件来知道你的偏好设置。

  1. 在项目文件夹里找到 config.example.yaml 文件，复制一份并改名为 config.yaml

  2. 用记事本打开 config.yaml

  3. 找到 openai_api_key 这一行，把你的 API Key 填进去

什么是 API Key？

API Key 就像一把"钥匙"，让工具能使用 AI 分析服务。你需要从 AI 服务商那里获取。

推荐选择（任选其一）：

  • DeepSeek（国内，便宜）：在 platform.deepseek.com 注册并创建 API Key
  • 通义千问（阿里云）：在 dashscope.aliyun.com 注册
  • OpenAI（国外）：在 platform.openai.com 注册

获取到 Key 后粘贴到配置文件即可。不要把 Key 分享给其他人。

如果你用的是国内的 AI 服务，还需要修改 openai_base_url，具体值参见附录。

第 5 步：首次运行

python run.py

小提示：首次运行需要以管理员身份打开命令提示符，这样工具才能读取微信的进程信息。方法：在搜索栏输入 cmd，右键点击"命令提示符"，选择"以管理员身份运行"。

第 1 步：安装 Python

Mac 自带的 Python 版本太旧，我们需要安装新版本。有两种方式：

方式 A：直接下载安装包（推荐新手）

  1. 访问 python.org，下载 macOS 版安装包

  2. 双击安装，一路点"继续"直到完成

方式 B：用 Homebrew 安装（更专业）

Homebrew 是 Mac 上的软件包管理工具，如果你还没装过，先打开"终端"（在启动台搜索"终端"或"Terminal"），粘贴以下命令：

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

然后安装 Python：

brew install python3

第 2 步：安装 SQLCipher

Mac 上需要额外安装一个数据库解密组件。在终端输入：

# 安装 SQLCipher
brew install sqlcipher

# 设置环境变量（让 Python 能找到 SQLCipher）
export LDFLAGS="-L$(brew --prefix sqlcipher)/lib"
export CPPFLAGS="-I$(brew --prefix sqlcipher)/include"

# 安装 Python 的 SQLCipher 接口
pip3 install pysqlcipher3

以 # 开头的行是注释，不需要输入。只输入不以 # 开头的那几行命令就行。

第 3 步：下载并安装证据助手

# 进入项目目录（假设放在桌面）
cd ~/Desktop/wechat_evidence_agent

# 安装所有依赖
pip3 install -e ".[all]"

第 4 步：配置

和 Windows 一样，复制配置文件并填写 API Key：

cp config.example.yaml config.yaml
open -e config.yaml

这会用 Mac 的文本编辑器打开配置文件，找到 openai_api_key 填入你的 Key。

第 5 步：密钥获取

Mac 比 Windows 多一个步骤：需要获取微信数据库的解密密钥。

工具会自动尝试从 macOS 钥匙串（Keychain）读取。如果自动读取失败，你需要手动设置：

# 方法一：设置环境变量（推荐）
export WECHAT_DB_KEY="你的密钥（十六进制字符串）"

# 方法二：运行工具时会提示你手动输入密钥

密钥只需要获取一次。获取方法可以参考开源项目 PyWxDump 的文档说明。

第 6 步：首次运行

python3 run.py

看到欢迎界面就说明安装成功了。

斜杠命令

除了自然语言对话，还有几个快捷命令（以 / 开头）：

查看帮助信息和所有可用命令

退出程序

重新开始对话（清除历史，但案件记忆保留）

快速导出当前已标记证据的清单文档

快捷命令行操作

如果你不想进入对话模式，也可以直接在命令行一行搞定：

# 提取某人聊天记录
python run.py extract --contact "张三" --start 2024-01-01 --end 2024-03-31

# 直接导出证据清单
python run.py export --contact "张三" --format docx --output ./output

# 搜索联系人
python run.py contacts --keyword "张"
04

证据清单生成

生成的文档长什么样

工具生成的 Word 文档严格遵循中国法院要求的证据清单格式：

  • 标题：居中的"证据清单"大字（黑体）
  • 案件信息：案由、原告、被告
  • 证据表格：包含以下列 —
序号证据名称证据来源内容摘要证明目的页码备注
1微信聊天记录微信 PC 客户端2024-01-15 张三发送"我先借你20万"证明借贷合意成立1-3
2微信转账记录微信 PC 客户端2024-01-16 向张三转账 ¥200,000证明款项已实际交付4
..................
  • 页脚：提交人、日期、页码
  • 字体：正文宋体、标题黑体（符合法院公文规范）

自定义案件信息

在 config.yaml 中可以预设案件信息，这样每次生成就不用重复输入：

evidence_template:
  court_name: "上海市浦东新区人民法院"
  case_number: "（2024）沪0115民初XXXX号"
  lawyer_name: "李律师"
  law_firm: "XX律师事务所"
  plaintiff: "王某某"
  defendant: "张某某"

当然也可以在对话中临时告诉 AI："原告是王某某，被告是张三，案号是..."

导出格式对比

格式优点适用场景
Word (.docx)可编辑，格式规范提交法院前还需要微调的情况（推荐）
PDF不可篡改，格式固定最终提交给法院
HTML包含交互元素，可嵌入图片内部查阅和归档
05

AI 分析功能

AI 能帮你做什么

  • 自动提取关键事实 — 从聊天记录中识别金额、日期、承诺、违约行为等法律要素
  • 识别争议焦点 — 自动判断双方的核心分歧在哪里
  • 评估证据完整性 — 告诉你证据链是否充分，哪些环节还缺证据
  • 匹配法律条文 — 将事实与相关法律条文自动关联（如民法典相关条款）
  • 建议诉讼策略 — 基于证据情况给出诉讼方向建议

AI 分析支持多种常见案由：

案由AI 重点分析内容
民间借贷纠纷借贷合意、款项交付、利息约定、还款期限、催告记录
合同纠纷合同成立、履行情况、违约行为、损失证据
劳动争议劳动关系确认、工资支付、解除/终止合同的合法性
房屋买卖/租赁合同条款、付款记录、交房/退房、违约金
婚姻家庭纠纷财产分配、抚养权相关事实、过错证据

AI 不能做什么

重要声明：AI 分析仅供参考，不能替代律师的专业判断。

AI 可能存在分析不准确的情况，所有分析结果请务必人工审核后再使用。

AI 不提供法律意见，也不能作为法律依据。

建议将 AI 的分析结果作为"辅助工具"，最终的策略决策由你来做。

常见问题

Q：运行后提示"找不到微信数据目录"

解决方法：

  • 确认微信 PC/Mac 版已登录并正在运行
  • Windows 默认路径：C:\Users\你的用户名\Documents\WeChat Files\
  • Mac 默认路径：~/Library/Containers/com.tencent.xinWeChat/
  • 如果你修改过微信的文件保存路径，在 config.yaml 中手动指定 wechat_dir

Q：提示"密钥提取失败"

Windows：需要以管理员身份运行命令提示符。右键 cmd → "以管理员身份运行"。

Mac：首次运行时系统可能会弹出"允许终端控制电脑"的权限请求，请点击允许。如果 Keychain 自动读取失败，参考第二章的手动密钥获取方法。

Q：AI 分析没有响应 / 报错

请检查：

  • config.yaml 中的 openai_api_key 是否正确填写
  • 如果用国内 AI 服务，openai_base_url 是否设置正确（参见附录）
  • 网络是否正常（如果用 OpenAI 需要能访问国际网络）
  • API 账户是否有余额

Q：语音消息没有转成文字

语音转文字需要额外安装 Whisper 模型。运行以下命令：

pip install openai-whisper

首次使用时会自动下载语音识别模型（约 140MB），需要耐心等待。

Q：生成的 Word 文档格式不对 / 乱码

证据清单使用宋体和黑体。如果你的电脑上没有这些字体：

  • Windows：一般自带，不需要额外安装
  • Mac：需要安装"宋体"和"黑体"字体，或者将文档发给 Windows 电脑打开

Q：提示 pip 找不到 / python 找不到

这是因为 Python 没有添加到系统 PATH。

Windows：卸载 Python 重新安装，这次一定要勾选 "Add Python to PATH"。

Mac：尝试使用 python3 和 pip3 代替 python 和 pip。

Q：安装 pysqlcipher3 失败

这个库需要系统先安装 SQLCipher 的 C 语言库。

Windows：最简单的方法是使用预编译的 wheel 文件安装，或者安装 Visual Studio Build Tools。

Mac：先运行 brew install sqlcipher，再设置环境变量（参见第二章 Mac 安装步骤）。

电子证据的法律效力说明

根据《民事诉讼法》第六十六条和《最高人民法院关于民事诉讼证据的若干规定》：

  • 微信聊天记录属于电子数据证据类型
  • 作为证据使用需满足真实性、关联性、合法性三个条件
  • 建议在条件允许时对电子证据进行公证保全，增强证明力
  • 本工具生成的证据清单可作为证据组织的辅助材料，但不能替代公证
  • 提交法院时建议同时保留微信原始界面的截图作为补充佐证

使用须知：本工具仅用于提取你自己微信账号的聊天记录。使用本工具获取、分析的数据必须合法合规，不得用于非法获取他人隐私信息。
